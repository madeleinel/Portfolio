    • Let's Paint

    A drawing application which allows the user to draw on a canvas and customise features such as brush stroke colour, size and effects, changing the background colour independently from the drawing itself, and undoing drawing actions.

    Tech Stack: ES6, HTML5 canvas and HTML5/CSS3.

    Website Source code

    Background

    The drawing application was inspired by Wes Bos’s JavaScript30 HTML5 canvas tutorial. The canvas and the basic drawing feature was set up using this tutorial, and all additional features have been added using ES6.

    More features are in the process of being added.

    Tech Stack

    ¬ HTML5 Canvas to set up the canvas

    ¬ ES6 to enable the drawing features

    ¬ HTML5 and CSS3 to style the site

    Challenge 1

    Enable the user to add brush effects.

    Solution

    Added event listeners to listen for changes within the effect list elements; when registering a change, it loops through the list to see which option element was chosen, and then sets the drawing effect to be the value of that option.

    Challenge 2

    Ability to change the background colour without overwriting the current drawing.

    Solution

    Added a secondary canvas below the primary one, and made the background of the primary canvas transparent.

    Challenge 3

    Enable the user to undo each drawing action.

    Solution

    When start each new drawing action, the system saves the last version of the site to an array. When clicking the Undo button, the canvas is restored to the top item of the array, and that item is then removed from that array to enable the user to undo all drawing actions one by one.

    Challenge 4

    Make the controls (the input elements) look nicer, while still displaying the current values (e.g. the current brush and background colours, the brush thickness, and the chosen brush effect.)

    Solution

    The default styling of the HTML input elements was overwritten using span elements. To enable the span elements to display the current values, ES6 was used to update the look of these elements each time the value is changed within the JavaScript code.

    Our Story

    A website created to provide a positive view of immigration in the UK.

    Winner of the Code First: Girls Introduction to Web Development course competition at The Guardian (2017).

    Tech Stack: HTML5/CSS3, Twitter API Bootstrap and jQuery.

    Website Source code

    Background

    The website was created as part of the Code First: Girls Introduction to Web Development course competition with E Jegede.

    Tech Stack

    ¬ jQuery for the interactive features

    ¬ HTML5, CSS3 and the Twitter API Bootstrap to style the site

    Challenge

    Make the user interact with the material and challenge their perceptions of immigrants.

    Solution

    Within the "Myths" section, the user is presented with common perceptions about immigrants and asked to guess whether the statement is true or false. Upon making their choice, a pop-up window appears which informs the user whether they were correct or not, and provides more information about the topic.

    WomenToTech

    A platform which matches self-identifying women who want to get started in the tech industry with mentors who are working within the industry.

    The platform database is currently being updated and more features are being added, so please be advised that the platform at times may not function properly.

    Tech Stack: Python 2, Flask, PostgreSQL, Twitter API, HTML5/CSS3, and Heroku.

    Website Source code

    Background

    The website was created as part of the Code First: Girls Advanced Python Web Development Course competition with E Keane.

    The platform database is currently being updated and more features, such as matching up mentors and mentees based on their preferred languages and what type of help they are looking for or can provide, are being added.

    Tech Stack

    ¬ Python 2 and Flask for backend

    ¬ PostgreSQL for database handling

    ¬ Twitter API to set up the Twitterbot

    ¬ HTML5 and CSS3 to style the site

    ¬ Heroku for app deployment

    Challenge 1

    Enable prospective mentors and mentees to use the platform.

    Solution

    Prospective mentors and mentees can sign up to the platform by tweeting us. When they sign up a profile is created for them on the website. They can then browse through the existing profiles on the website.

    Challenge 2

    Create profiles on the website containing the information provided through the sign-up tweet.

    Solution

    When the user tweets us, the information they have provided is saved into our database using the tags used within the tweet (eg "-langs" and "-skills"). The content of the database is then imported into the profile pages on the website. A for loop ensures that all new users are displayed as they sign up.
